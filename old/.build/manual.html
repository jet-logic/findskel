Name

fileop.py — File search and operate. Add your own module for extra/extended/customized operation

Synopsis

python fileop.py [--empty] [--no-empty] [--type CODE] [--carryon] [--maxdepth N] [--mindepth N] [ -L | --follow ] [ -d | --depth ] [-g GLOB...] [-e PATTERN...] [--del] [ -s <MATCH><REPLACE> | --ren <MATCH><REPLACE> ...] [--stat] [--exstat] [ -p MODULE | --package MODULE ] [--basename] [--mingw] [--cygwin] [--uri] [--rel-to DIR] [--version] [--dry-run] [--act] PATH...

Description

fileop.py utility to search for files and directories using wildcard and/or regular expression. Filter by type, date, attributes, and size. And do operations like move, copy, delete and rename on matched files.

Arguments

Main option:

--version

Show version.

--dry-run

Perform a trial run with no changes made.

--act

Depends on operation. May modify file.

PATH

The directory to search

Directory traversal option:

--carryon

Process next file even if error occurs.

--maxdepth N

Recurs directories only N level deep.

--mindepth N

Process only files under N level deep.

-L, --follow

Follow symbolic links.

-d, --depth

Process each directory's contents before the directory itself.

-g GLOB

File name glob. Example “read*.txt” for all text files whose filename starts with “read”

-e PATTERN

File name regular expression pattern. Example “read.+\.txt”

Filter files:

--empty

Process only empty files or directories.

--no-empty

Process only non-empty files or directories.

--type CODE

Filter the file type. See the section called “Filter files base on type” in manual.

Operation option:

--del

Delete found files. Use --act to actually delete the file.

-s <MATCH><REPLACE>, --ren <MATCH><REPLACE>

Substitute file names to this regex pattern. This is similar to perl's substitution operator (s{}{}). The bracket quote ([], {}, <>, ()) must be balance. Use --act to rename the file.

--stat

Show statistics. The number of files, folders, total size.

--exstat

Show statistics about file extension. The number of files using a certain file extension.

-p MODULE, --package MODULE

MODULE is module import name or the actual module file path to load. See the section called “User modules” in manual.

Pretty print options:

--basename

Print only name portion of path.

--mingw

Print path in MingW path format. For MS Windows only.

--cygwin

Print path in CygWin path format. For MS Windows only.

--uri

Print path in URI path format. e.g. file:///B:/app/README

--rel-to DIR

Print path relative to DIR

Filter files base on type

Use --type CODE option to filter file type.CODE must be one or more of the following characters:

b block file
c character file
d directory file
f regular file
s symlink file
p fifo file
s socket file

“-” Prefix meaning not.To list all the regular files in the current directory:

fileop --type f .

To list all symbolic link and regular files in the current directory:

fileop --type fs
